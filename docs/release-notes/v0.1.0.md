# Squirix 0.1.0 — first public preview

**Release status:** Early preview — not production-ready.

squirix 0.1.0 is the first public preview focused on validating the client/server architecture, typed cache API, and
durability foundation. The goal of this release is API, architecture, and operational feedback from .NET backend and
distributed-systems engineers evaluating cache infrastructure libraries.

NuGet package version: **`0.1.0-preview.4`** (latest preview)

NuGet package ids use lowercase **`squirix.*`** (`squirix`, `squirix.server`, `squirix.server.tool`). The earlier
PascalCase ids (`Squirix`, `Squirix.Server`, `Squirix.Server.Tool`) are deprecated on nuget.org.

## 0.1.0-preview.4

- Publish `squirix.server.tool` global tool to nuget.org
- `Dockerfile.dev` (build from sources) and `Dockerfile.release` (NuGet global tool)
- Fix Docker Compose cluster settings (`Cluster.Url` must match local peer entry)
- Single primary HTTPS listener (gRPC, `/health`, `/metrics`); removed plaintext HTTP sidecar
- Non-loopback listen URLs require JWT at startup (no unauthenticated external opt-out)
- Docker images bundle a development TLS PFX for Kestrel (`ASPNETCORE_Kestrel__Certificates__Default__*`)
- Documentation: tool install, dual Dockerfiles, container quick start, HTTPS Docker notes

## 0.1.0-preview.3

- Documentation proofread: README quick start, package table, naming alignment
- Release workflow tolerates duplicate symbol package uploads on rerun
- Publish `squirix` and `squirix.server` to nuget.org from the release workflow

## 0.1.0-preview.2

- Renamed NuGet package ids to lowercase `squirix`, `squirix.server`, and `squirix.server.tool`
- First lowercase-id publish to nuget.org (`squirix`)

## Highlights

- Strict **client/server** package boundary (`Squirix` client SDK + server runtime)
- Strongly typed **`ICache<T>`** with explicit read result types
- **gRPC** application transport
- Per-node **write-ahead journal**, **snapshots**, **compaction**, and **recovery**
- **Health**, **readiness**, and **admin** HTTP endpoints
- **Prometheus** text metrics scrape (`/metrics`) and server-side **OpenTelemetry** journal tracing
- Standalone **`squirix-server`** CLI and Docker sample for local evaluation
- Static cluster topology with consistent-hash **single-owner routing**

## Install

Client SDK:

```powershell
dotnet add package squirix --version 0.1.0-preview.4
```

Server runtime library:

```powershell
dotnet add package squirix.server --version 0.1.0-preview.4
```

Standalone server host (global tool):

```powershell
dotnet tool install --global squirix.server.tool --version 0.1.0-preview.4
squirix-server run --data-dir ./data
```

Docker (`Dockerfile.dev` from sources, `Dockerfile.release` from NuGet tool):

```powershell
docker build -f Dockerfile.dev -t squirix-server .
docker run --rm `
  -p 5000:5000 `
  -e SQUIRIX_JWT_SIGNING_KEY=dev-squirix-docker-jwt-key!!!!!! `
  -e SQUIRIX_JWT_ISSUER=https://squirix.docker.dev `
  -e SQUIRIX_JWT_AUDIENCE=squirix `
  squirix-server run --urls https://0.0.0.0:5000

docker build -f Dockerfile.release -t squirix-server:0.1.0-preview.4 .
```

Images ship a bundled development HTTPS certificate; use `curl -k` from the host. Docker examples configure JWT env
vars — pass a matching bearer token via `options.BearerTokenProvider` or `Authorization: Bearer`.

Requires **.NET 10**.

## Quick usage

```csharp
using System.Threading;
using Squirix;

var cancellationToken = CancellationToken.None;

// Local tool default (no JWT unless you configure auth):
await using var client = await SquirixClient.ConnectAsync("https://localhost:5001", cancellationToken);

// Docker examples: use https://localhost:5000 (or :5001 for Compose node A), options.BearerTokenProvider,
// and a development TLS validation override — see containerization.md.

var cache = await client.GetCacheAsync<string>("demo", cancellationToken);
await cache.SetAsync("greeting", "hello", cancellationToken: cancellationToken);

var lookup = await cache.GetValueAsync("greeting", cancellationToken);
if (lookup.Found)
{
    Console.WriteLine(lookup.Value);
}
```

Squirix gRPC transport requires HTTPS endpoints. Trust the ASP.NET Core development certificate for local non-Docker
work (`dotnet dev-certs https --trust`). Docker images use their own bundled development PFX; see
[containerization.md](../containerization.md).

## Known limitations

- **Experimental preview** — API, wire format, and on-disk layouts may change during 0.x
- **Not production-ready** — durability, recovery, and performance characteristics are still hardening
- **No replication or automatic failover** — durability is per node
- **Static peer configuration** — no dynamic membership or runtime topology changes in v0.1
- **Single-key scope** — no cluster-wide transactions; cross-key atomicity is out of scope
- **Narrow client surface** — basic KV + expiration only; no batch, scan, watch, or tag invalidation yet
- **.NET 10 only** — no multi-TFM support in this preview line
